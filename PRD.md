# 幼儿英语启蒙 App 产品需求文档 (PRD)

**版本：** 2.2
**核心哲学：** 1. **直觉交互：** 孩子不需要学习如何使用 App。
2. **零翻译：** 建立“图像=声音”的直接联系。
3. **因果一致：** 所有的点击动作必须产生可预测的、愉悦的反馈。

1. 产品基本信息

* **产品名称：** HopEnglish
* **产品定位：** 一款面向 2-5 岁幼儿的**沉浸式“因果反馈”英语启蒙工具**。
* **核心理念：** 剥离翻译，回归本能。利用幼儿对“动作-反馈”的天然敏感，建立声音与图像的直接映射，模拟母语习得环境。

2. 用户分析 (User Profile)

* **核心用户：** 2-5 岁处于语言发育黄金期的幼儿。
* **用户特征：**
* **非识字：** 无法阅读中文字符或英文字母。
* **形象思维：** 依赖具体的、高饱和度的图像理解世界。
* **专注力短：** 需要极强的即时正向反馈（声音、动效）来维持兴趣。
* **交互本能：** 喜欢“点按”和“重复”。

## 零、 第一性原理与学习闭环（所有需求的出发点）

### 1. 第一性原理（2-5 岁孩子如何“学会”）

* **孩子不是先理解再行动，而是先行动再形成规则：** 动作 → 反馈 → 强化（重复）。
* **抽象弱、对比强：** 只能稳定理解“可看见/可变化”的属性（大/小、红/蓝、跑/停）。
* **注意力短、工作记忆小：** 单次任务必须“1 步完成”，界面上可选项越少越好。
* **听觉理解先于发音输出：** 不把“跟读/说准”作为学习门槛，输出应为可选玩法。
* **记忆靠高频复现与泛化：** 同一声音需要在不同画面/例子中出现，避免“只记一张图”。

### 2. 本产品的最小体验闭环（设计目标）

* **闭环目标 A（直觉联想）：** 听到发音时，孩子自然把它和眼前的图像/变化联系在一起。
* **闭环目标 B（自发重复）：** 孩子愿意反复点击触发同一反馈（“再来一次”）。
* **闭环目标 C（不过度依赖单一图片）：** 同一个声音可以对应多种同类图像/场景（避免“只记一张图”）。
* **闭环目标 D（可选模仿）：** 孩子在高动机时自发模仿声音即可，不设任何门槛与流程。

### 3. 关键产品规则（从第一性原理推导）

* **单一任务原则：** 一个界面只让孩子做一件事（点击图片/点击下一个/切换状态）。
* **零解释原则：** 不依赖文字说明、教程、弹窗；孩子“试一下就会”。
* **即时反馈原则：** 点击后必须立刻出现视觉反馈（Q 弹/闪光/粒子），声音紧随其后。
* **可预测性原则：** 同样的动作永远产生同样的结果（避免随机导致困惑）。
* **无失败态原则：** 没有红叉、扣分、失败提示；点错也要温和反馈且不跳出。
* **重复即奖励：** 重复点击不是“刷”，而是学习路径；系统应鼓励重复。

### 2. 大图学习仓 (Learning Core) —— 核心页面

* **需求描述：** 沉浸式展示单词，通过点击和按钮顺序浏览。
* **页面目标：** 把英语变成"声音-意义玩具"：孩子一点击就得到稳定、愉悦、可重复的反馈。

**单词顺序策略（会话内固定，会话间洗牌）：**

| 时机 | 行为 |
|------|------|
| 进入主题时 | 对单词列表进行一次随机洗牌 |
| 本次会话内 | 顺序完全固定，Next 永远是"下一个" |
| 下次进入时 | 重新洗牌，生成新的顺序 |

**设计理由（第一性原理）：**

| 问题 | 分析 |
|------|------|
| 纯固定顺序 | ❌ 短 session 导致前几个词曝光过高，后几个词长期冷落 |
| 纯随机乱序 | ❌ 每一步都随机会破坏规则感，降低重复动机 |
| 会话内固定+会话间洗牌 | ✅ 规则稳定（孩子能学会"点下一个就是下一个"）+ 覆盖均匀（每个词都有机会出现在开头）|

**（补充）学习数据记录与自适应排序（vNext）：**

> 目标：在不改变孩子端交互规则（仍然“会话内固定”）的前提下，利用“看到/听到 + 时间”让覆盖更均匀、复习更及时、偏好更被尊重。

**A. 数据口径（系统内部记录，不对孩子展示）**

| 指标 | 含义 | 触发时机（建议默认） | 备注 |
|------|------|----------------------|------|
| 看到次数 `viewCount` | 单词成为当前展示词的次数 | 进入主题展示第一个词；Next 切换到新词；目录跳转到某词 | 防误触：若停留过短（如 < 300–500ms）可不计 |
| 听到次数 `playCount` | 单词音频被触发播放的次数 | 进入主题自动播放；点击卡片重播；切到新词自动播放 | 防连点噪声：同词短时间内重复触发（如 800–1200ms）可合并计 1 次 |
| 最后看到时间 `lastSeenAt` | 最近一次“看到”该词的时间 | `viewCount` +1 时更新 | 用于“间隔复习/久别重启”判断 |

**B. 会话（session）定义**

| 规则 | 定义 |
|------|------|
| 会话范围 | 从进入某主题学习页开始，到离开该页结束 |
| 中断恢复 | 切后台/锁屏后在短时间内返回（如 ≤30 分钟）仍算同一会话；超过阈值视为新会话 |

**C. 时间因素（第一性原理：记忆痕迹随时间衰减）**

按“主题维度”判断（某主题隔多久没学），将进入主题时的复习策略分档：

| 距离上次学习该主题的间隔 | 模式 | 行为倾向 |
|--------------------------|------|----------|
| < 2 天 | 日常巩固 | 覆盖更均匀，少量复习穿插 |
| 2–7 天 | 复习加强 | 复习比例更高，避免陌生感过强 |
| ≥ 7 天 | 久别重启 | 先熟后新：先用“熟悉/偏好词”恢复规则感与动机，再进入复习与覆盖 |

**D. 自适应排序原则（会话内固定，会话间自适应）**

1. **生成时机**：进入主题时生成“本次会话的完整顺序”，会话内不再变化。
2. **优先级**：
   - **覆盖/复习（viewCount + lastSeenAt）为主**：避免某些词长期“饿死”。
   - **偏好（playCount）为辅**：尊重孩子主动重复，但不让偏好劫持整体覆盖。
3. **推荐落地策略：分桶 + 混排（稳定、可控、易解释）**

将词分为 3 个桶（桶内轻随机）：

| 桶 | 目标 | 典型判定（示例） |
|----|------|------------------|
| Overdue（需要复习） | 复习与找回 | `daysSince(lastSeenAt)` 大（如 ≥7）或 `viewCount` 很低（如 ≤1） |
| Warm（常规巩固） | 平滑巩固 | 最近见过且次数中等 |
| Favorite（兴趣奖励） | 维持动机 | `playCount - viewCount` 较高（孩子主动重复多） |

**排序配比（以“前 5 个最关键位”为设计目标，适配短 session）：**

| 模式 | 前 5 个建议构成 |
|------|------------------|
| 日常巩固 | Overdue 2 + Warm 3（最多穿插 1 个 Favorite） |
| 复习加强 | Overdue 3 + Warm 2 |
| 久别重启 | Favorite/熟悉 2 + Overdue 2 + Warm/新 1 |

> 说明：前几张卡片最关键，因为幼儿常出现“只玩前几个就退出”的短会话行为；将复习与覆盖优先放在前段，能显著提升长期均匀性。

**UI 示意图：**

```
┌──────────────────────────────────────────┐
│                                          │
│   ←   🦁 Animals                    📋   │  ← Header: 返回 | 主题 | 目录
│                                          │
├──────────────────────────────────────────┤
│                                          │
│                                          │
│                  🦁                      │  ← 大图展示区
│                                          │     点击 = Q弹动画 + 发音
│                 Lion                     │
│                                          │
│                                          │
│                                          │
├──────────────────────────────────────────┤
│                                          │
│            ┌───────────────┐             │
│            │   下一个 →    │             │  ← 底部操作区
│            └───────────────┘             │
│                                          │
└──────────────────────────────────────────┘
```

* **页面布局：**
  * **顶部 Header：** 返回按钮 | 分类图标+名称 | 目录按钮
  * **视觉中心：** 大尺寸、高清插图，点击区域
  * **底部固定位：** 巨大的"下一个 (Next)" 按钮

* **交互逻辑：**
  1. **点击图片：** 产生 Q 弹动画 → 播放单词发音（如：`Lion`）
  2. **点击"下一个"：** 切换到本分类的下一个单词
  3. **重复播放：** 允许连续点击图片重复播放，鼓励重复
  4. **按键冷却：** "下一个"按钮 0.5 秒冷却，防止连点乱跳
  5. **完成分类：** 浏览完最后一个单词后，进入庆祝页（见下节）

### 3. 庆祝页 (Celebration Page)

* **需求描述：** 完成一个分类的学习后，展示独立的庆祝页面。
* **页面目标：** 给予孩子积极的视觉/听觉反馈，强化"完成=快乐"的心理联结。
* **设计理由：** 独立页面提供明确的"场景切换"，比覆盖层更有仪式感。

**UI 示意图：**

```
┌──────────────────────────────────────────┐
│                                          │
│    🦁  ✨  🐕  🎉  🐈  🌟  🐦  ⭐  🐟    │  ← 撒花粒子
│                                          │
│              （撒花动画）                 │
│                                          │
│                                          │
│            ✨ Well done! ✨              │  ← 中央大字（弹出动画）
│                                          │
│                                          │
│         🔊 播放 "Well done!" 语音        │
│                                          │
│                                          │
│            ┌────────────┐                │
│            │  Done (5)  │                │  ← Done 按钮 + 倒计时
│            └────────────┘                │
│                                          │
└──────────────────────────────────────────┘
```

* **页面元素：**
  * **撒花动画：** 从底部喷射学过的单词 emoji + 装饰 emoji（🎉✨🌟⭐🎊），形成扇形后缓缓散落
  * **中央文字：** "Well done!" 大字，弹出动画效果（0 → 1.2 → 0.9 → 1.0）
  * **语音表扬：** 固定播放 "Well done!" 语音
  * **Done 按钮：** 全宽按钮，样式与学习页 Next 按钮一致
  * **倒计时：** 按钮旁显示 (5) → (4) → (3) → (2) → (1)
  * **渐变背景：** 根据主题色生成，与学习页区分

* **交互逻辑：**
  1. 进入页面 → 撒花开始 + 播放语音 + 文字弹出
  2. 点击 Done → 立即返回首页
  3. 不操作 → 5 秒后自动返回首页

* **第一性原理检验：**
  * 孩子视角："Well done!" 文字和倒计时**孩子看不懂**，但**无害**
  * 核心反馈（撒花、语音、按钮）服务孩子
  * 附加信息（文字、倒计时）服务**陪读家长**

## 三、 用户体验与反馈规范

### 1. 声音反馈

* **标准：** 纯正童声或柔和女声。
* **逻辑：** 进入页面自动读一遍，再次点击图片重读。
* **一致性：** 同一词/短语的音频版本必须稳定（避免每次随机音色导致混淆）。
* **节奏：** 发音尽量短、清晰、带轻微童趣但不过度夸张。

**表扬语音（已实现）：**

| 文件 | 内容 | 场景 |
|------|------|------|
| `great.wav` | "Great!" | 完成分类庆祝（备用） |
| `well_done.wav` | "Well done!" | 完成分类庆祝（**当前使用**） |
| `good_job.wav` | "Good job!" | 完成分类庆祝（备用） |
| `awesome.wav` | "Awesome!" | 完成分类庆祝（备用） |
| `yay.wav` | "Yay!" | 完成分类庆祝（备用） |

* **语调要求：** 兴奋欢快，像游戏通关后的庆祝
* **播放策略：** 固定播放 `well_done.wav`（"Well done!"），保持一致性

### 2. 激励反馈

* **完成庆祝（已实现）：** 完成一个主题后，进入独立庆祝页面：
  * **视觉：** 从底部喷射学过的单词 emoji + 装饰 emoji（🎉✨🌟⭐🎊），形成扇形后缓缓散落
  * **文字：** 中央显示 "Well done!" 大字，弹出动画效果（给陪读家长的视觉反馈）
  * **听觉：** 固定播放 "Well done!" 表扬语音
  * **按钮：** 底部 Done 按钮 + 倒计时 (5→4→3→2→1)，样式与学习页 Next 按钮一致
  * **结束：** 点击 Done 立即返回首页，或 5 秒后自动返回

* **补充原则：**
  * 激励反馈服务"重复动机"，不做复杂积分/排行榜（对 2-5 岁意义不大且易分心）
  * 不出现"失败惩罚"；完成主题的撒花是"庆祝"，不是"考试结算"
  * 撒花粒子使用学过的单词图像，强化记忆联结

### 3. 容错设计

* **按键锁定：** 防止连点导致的页面疯狂跳动。设置 0.5 秒的按钮冷却时间。
* **强制横屏：** 模拟绘本阅读体验，提供更大的操作空间。

### 4. 护眼休息（系统默认，玩具自带规则）

* **目标：** 避免长时间连续刺激，保护注意力与视疲劳。
* **规则：** 连续使用达到 15 分钟触发一次“休息页”（后续可扩展为 30 分钟档，但默认先做 15 分钟）。
* **休息页形态：** 柔和动画 + 简单倒计时；不出现“惩罚/责备”文字；倒计时结束自动返回到上次页面。

## 四、 内容体系（从“生活可迁移”出发）

### 1. 内容优先级（不是只做名词）

* **名词（可指认）：** apple、dog、car…
* **属性（可对比）：** big/small、red/blue、happy/sad…
* **动作（可观察）：** run、jump、sleep、open/close…
* **高频词块（强迁移）：** again、more、stop、hello、bye…

### 2. 核心主题规划 (MVP)

基于幼儿认知发展顺序，首批上线以下 7 个高频生活主题：

| 主题 | 核心词 |
|------|--------|
| 🦁 **Animals** | Lion, Dog, Cat, Bird, Fish |
| 🍎 **Foods** | Apple, Banana, Milk, Cookie, Water |
| 🚗 **Vehicles** | Car, Bus, Train, Plane, Boat |
| 🏃 **Actions** | Run, Jump, Sleep, Eat, Dance |

| 🌤️ **Nature** | Sun, Moon, Star, Rain, Flower |

### 3. 内容颗粒度与数量建议（MVP）

* 每个主题 5 个核心词（保持简洁，鼓励重复）
* 共计 35 个单词
* 后续版本可扩展到每个主题 12-20 个词

## 五、 后续版本规划

### 1. 形容词学习（v2.0 - 需重新设计）

**原"魔术棒"方案的问题（第一性原理分析）：**

| 原则 | 问题 |
|------|------|
| 单一任务原则 | ❌ 需要 3 步操作：点图片 → 点魔术棒 → 再点图片 |
| 认知负担 | ❌ 需理解"名词+形容词=短语"的组合关系 |
| 年龄适配 | ❌ 2-3 岁难以理解，仅适合 4-5 岁 |
| 复现稀释 | ❌ 可能分散对基础单词的重复 |

**替代方案（待验证）：**

| 方案 | 说明 | 符合原则 |
|------|------|----------|
| **对比卡片** | 同屏展示两张卡片（Big Apple vs Small Apple） | ✅ 一步对比 |
| **形容词独立教学** | 把 Big/Small/Fast/Slow 作为独立单词 | ✅ 单一任务 |
| **动态变化** | 图片自动变化，无需点击魔术棒 | ✅ 减少步骤 |

**决策：** MVP 不做，待核心功能验证后重新评估。

### 6. 自适应复习与内容调度（系统内部）

**功能描述：** 基于每个单词的“看到次数/听到次数 + 最后学习时间”，在进入主题时生成本次固定顺序，用于优化覆盖均匀性与复习时机（不改变孩子端交互，不展示统计数据）。

**价值：**
- **覆盖更均匀**：减少“只学到前几个词”的结构性偏置
- **复习更及时**：隔久未学时优先找回关键联结
- **动机更强**：保留孩子偏好词作为奖励穿插，鼓励重复点击
