# 启动页设计说明

## 设计概述

HopEnglish 启动页采用简洁、友好的设计风格，符合应用的整体品牌形象。

## 设计元素

### 1. 背景色
- **颜色**: 奶油白 `#FFF9F0`
- **说明**: 与应用图标背景色一致，营造温暖、舒适的视觉感受，实现无缝过渡

### 2. 布局结构
- **布局方式**: 完全居中（水平和垂直居中）
- **对齐方式**: 居中对齐

### 3. 应用图标
- **尺寸**: 180x180 像素
- **位置**: 屏幕正中央（水平和垂直居中）
- **资源**: `LaunchLogo`（来自应用图标）
- **显示模式**: Scale Aspect Fit（保持比例）
- **设计**: Q版狮子头，暖橙色，与图标背景（奶油白）完美融合

## 设计原则

1. **极简**: 只显示应用图标，无多余元素
2. **一致**: 背景色与图标背景色一致（奶油白），实现无缝视觉过渡
3. **友好**: 温暖的背景色和可爱的 Q 版狮子图标
4. **快速**: 极简设计确保快速加载和显示

## 技术实现

### 文件位置
- **Storyboard**: `ios/Runner/Base.lproj/LaunchScreen.storyboard`
- **图标资源**: `ios/Runner/Assets.xcassets/LaunchLogo.imageset/`

### 关键配置
- 使用 Auto Layout 实现响应式布局
- 支持所有 iPhone 尺寸（包括安全区域）
- 图标资源支持 @1x, @2x, @3x 分辨率

## 注意事项

1. **启动页限制**: iOS 启动页必须是原生 Storyboard，不能使用 Flutter 代码
2. **字体限制**: Storyboard 不支持自定义字体（Fredoka），使用系统字体作为替代
3. **动画限制**: 启动页不支持动画，只能显示静态内容
4. **显示时长**: 启动页显示时间由系统控制，通常很短（< 1 秒）

## 未来优化建议

如果需要更丰富的启动页体验，可以考虑：

1. **Flutter 启动页**: 在应用启动后立即显示一个 Flutter 实现的启动页
   - 可以使用自定义字体（Fredoka）
   - 可以添加简单的动画效果
   - 可以显示加载进度

2. **品牌元素增强**: 
   - 添加品牌色彩点缀
   - 使用渐变背景
   - 添加装饰性元素
